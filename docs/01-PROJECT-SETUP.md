# 01 — Project Setup (Milestone 1)

> From an empty root to a running backend + frontend with health check,
> professional tooling, Docker, and CI-ready infrastructure.

## 1. What This Milestone Delivers

By the end of this document, the `main` branch contains:

- A Python backend (FastAPI) that serves a `/api/health` endpoint
- A React frontend (Vite + TypeScript + Tailwind) that renders a landing page and shows the health status
- FastAPI serves the frontend's built static files in production
- SQLite database connection (aiosqlite + SQLAlchemy async) with Alembic migrations
- Redis connection verification on startup
- Docker multi-stage build producing a single image
- `docker-compose.yml` that starts Sential + Redis with one command
- Linting (ruff), type checking (pyright), formatting (ruff format), testing (pytest)
- Frontend linting (eslint), formatting (prettier), testing (vitest)
- A `.gitignore` that covers both Python and Node
- A `Makefile` with common development commands
- A `README.md` with setup and run instructions

Nothing else. No ingestion, no agents, no graph logic. Just the skeleton, wired end-to-end, running cleanly.

### 4.2 Backend: Configuration (`backend/config.py`)

All configuration flows through a single `Settings` class using `pydantic-settings`. Every value comes from environment variables (or `.env`).

```python
from pathlib import Path
from pydantic_settings import BaseSettings, SettingsConfigDict


class Settings(BaseSettings):
    model_config = SettingsConfigDict(
        env_prefix="SENTIAL_",
        env_file=".env",
        env_file_encoding="utf-8",
        extra="ignore",
    )

    data_dir: Path = Path("./data")
    redis_url: str = "redis://localhost:6379"
    host: str = "0.0.0.0"
    port: int = 8000
    log_level: str = "info"
    database_url: str = ""

    def get_database_url(self) -> str:
        if self.database_url:
            return self.database_url
        db_path = self.data_dir / "sential.db"
        return f"sqlite+aiosqlite:///{db_path}"


settings = Settings()
```

### 4.3 Backend: Database (`backend/database.py`)

Async SQLAlchemy engine with session factory. One engine for the app lifetime, sessions created per-request.

```python
from collections.abc import AsyncGenerator
from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker, create_async_engine
from backend.config import settings


engine = create_async_engine(
    settings.get_database_url(),
    echo=False,
    connect_args={"check_same_thread": False},
)

async_session_factory = async_sessionmaker(engine, expire_on_commit=False)


async def get_session() -> AsyncGenerator[AsyncSession, None]:
    async with async_session_factory() as session:
        yield session
```

### 4.5 Backend: Health Endpoint (`backend/api/health.py`)

The health check verifies three things: the API is up, SQLite is reachable, Redis is reachable.

```python
from fastapi import APIRouter, Depends
from sqlalchemy import text
from sqlalchemy.ext.asyncio import AsyncSession
from redis.asyncio import Redis

from backend.config import settings
from backend.database import get_session

router = APIRouter(prefix="/api", tags=["health"])


@router.get("/health")
async def health(session: AsyncSession = Depends(get_session)) -> dict:
    status = {"api": "ok", "database": "ok", "redis": "ok"}

    try:
        await session.execute(text("SELECT 1"))
    except Exception:
        status["database"] = "error"

    try:
        redis = Redis.from_url(settings.redis_url)
        await redis.ping()
        await redis.aclose()
    except Exception:
        status["redis"] = "error"

    healthy = all(v == "ok" for v in status.values())
    return {"healthy": healthy, **status}
```

### 4.6 Backend: App Factory (`backend/main.py`)

The FastAPI app with lifespan management (creates data directory, runs migrations on startup), static file serving for the frontend, and CORS for development.

```python
from contextlib import asynccontextmanager
from collections.abc import AsyncIterator
from pathlib import Path

from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.staticfiles import StaticFiles

from backend.config import settings
from backend.api.health import router as health_router


@asynccontextmanager
async def lifespan(app: FastAPI) -> AsyncIterator[None]:
    settings.data_dir.mkdir(parents=True, exist_ok=True)
    yield


app = FastAPI(title="Sential", version="1.0.0", lifespan=lifespan)

app.add_middleware(
    CORSMiddleware,
    allow_origins=["http://localhost:5173"],  # Vite dev server
    allow_methods=["*"],
    allow_headers=["*"],
)

app.include_router(health_router)

static_dir = Path(__file__).parent / "static"
if static_dir.is_dir():
    app.mount("/", StaticFiles(directory=static_dir, html=True), name="static")
```

### 4.8 Backend: Alembic Setup

Initialize Alembic:

```bash
uv run alembic init backend/migrations
```

Then edit `alembic.ini` — set the SQLAlchemy URL to a placeholder (the real URL comes from `env.py`):

```ini
# alembic.ini (relevant line)
sqlalchemy.url = sqlite+aiosqlite:///./data/sential.db
```

Replace `backend/migrations/env.py` with an async-aware version that imports `Base.metadata`:

```python
import asyncio
from logging.config import fileConfig

from alembic import context
from sqlalchemy.ext.asyncio import create_async_engine

from backend.config import settings
from backend.models.base import Base

config = context.config
if config.config_file_name is not None:
    fileConfig(config.config_file_name)

target_metadata = Base.metadata


def run_migrations_offline() -> None:
    url = settings.get_database_url()
    context.configure(url=url, target_metadata=target_metadata, literal_binds=True)
    with context.begin_transaction():
        context.run_migrations()


def do_run_migrations(connection) -> None:
    context.configure(connection=connection, target_metadata=target_metadata)
    with context.begin_transaction():
        context.run_migrations()


async def run_migrations_online() -> None:
    connectable = create_async_engine(settings.get_database_url())
    async with connectable.connect() as connection:
        await connection.run_sync(do_run_migrations)
    await connectable.dispose()


if context.is_offline_mode():
    run_migrations_offline()
else:
    asyncio.run(run_migrations_online())
```

#### `frontend/src/api/client.ts`

A minimal typed fetch wrapper. All API calls go through this.

```typescript
const BASE = "/api";

export async function fetchHealth(): Promise<{
  healthy: boolean;
  api: string;
  database: string;
  redis: string;
}> {
  const res = await fetch(`${BASE}/health`);
  if (!res.ok) throw new Error(`Health check failed: ${res.status}`);
  return res.json();
}
```

Two terminal workflow. The backend and frontend run as separate processes during development.

**Terminal 1 — Backend:**

```bash
# First time only:
cp .env.example .env
# Edit .env if needed (Redis URL, etc.)

# Start Redis (if not using Docker for Redis):
docker run -d --name sential-redis -p 6379:6379 redis:7-alpine

# Run the backend:
make dev-backend
```

The API is now at `http://localhost:8000`. Test it:

```bash
curl http://localhost:8000/api/health
```

Expected response:

```json
{ "healthy": true, "api": "ok", "database": "ok", "redis": "ok" }
```

**Terminal 2 — Frontend:**

```bash
make dev-frontend
```

The UI is now at `http://localhost:5173`. It proxies `/api/*` to the backend. Open the browser and see the landing page with the green health indicator.

## 9. Dependency Versions (Pinning Strategy)

**Python dependencies** in `pyproject.toml` are specified **without version pins** (e.g., `"fastapi[standard]"`). The `uv.lock` file is the lockfile and contains the exact resolved versions. This is the standard uv workflow: `pyproject.toml` declares intent, `uv.lock` guarantees reproducibility.

**Node dependencies** in `frontend/package.json` follow the same pattern: semver ranges in `package.json`, exact versions in `package-lock.json`.

**Docker base images** are pinned to major versions (`python:3.12-slim`, `node:22-alpine`, `redis:7-alpine`) for stability without over-constraining patch updates.

## 10. Key Design Notes

**Why no Alembic migration yet?** There are no ORM models beyond `Base`. The first real migration happens in Milestone 2 when we add the `files` and `repositories` tables. The Alembic setup is in place and ready.

**Why aiosqlite?** FastAPI is async. Synchronous SQLite calls would block the event loop. `aiosqlite` wraps SQLite in a thread pool, making it compatible with async/await. SQLAlchemy's async engine handles this transparently.

**Why Redis in Milestone 1?** It's an external dependency that must be present from day one (arq job queue depends on it in Milestone 4). Including it now means the Docker setup is complete from the start and the health check validates the full infrastructure.

**Why TanStack Query on the frontend?** It handles caching, background refetching, and loading/error states out of the box. Every future API call will go through it. Setting it up now avoids retrofitting later.

**Why proxy in vite.config.ts?** During development, the frontend runs on `:5173` and the backend on `:8000`. The proxy ensures the frontend calls `/api/health` without CORS issues or hardcoded URLs. In production, both are served from the same origin (FastAPI serves the static files), so no proxy is needed.

**Why CORS middleware?** During development, if the developer opens the frontend directly at `:5173` without the proxy working (e.g., testing from a mobile device), CORS headers prevent browser blocking. In production, it's same-origin so CORS is irrelevant.

**What about the `old/` directory?** It stays in the repo for reference during the rewrite. Once Milestone 7 (remaining languages) is complete, the `old/` directory can be removed.
